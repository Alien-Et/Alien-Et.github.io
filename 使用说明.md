# Hexo网站使用说明

## 一、文章添加方法

### 1. 创建新文章

**方法一：使用 Hexo 命令**
```bash
# 在项目根目录执行
npx hexo new "文章标题"

# 这会在 source/_posts/ 目录下创建一个新的 Markdown 文件
# 文件名格式：YYYY-MM-DD-文章标题.md
```

**方法二：手动创建文件**
- 在 `source/_posts/` 目录下创建 `.md` 文件
- 文件名建议使用英文或拼音，如 `solar-panel-launch.md`
- 文件开头添加 Front Matter（前置数据）

### 2. 文章格式

每个文章文件开头需要添加以下 Front Matter：

```yaml
---
title: 文章标题
date: 2024-12-01 00:00:00
categories: [分类1, 分类2]
tags: [标签1, 标签2]
description: 文章描述
cover: /images/posts/图片文件名.svg
---

文章内容...
```

**Front Matter 说明：**
- `title`：文章标题（必填）
- `date`：发布日期（必填）
- `categories`：文章分类（可选）
- `tags`：文章标签（可选）
- `description`：文章描述（可选）
- `cover`：文章封面图片路径（可选）

### 3. 文章内容编写

**支持 Markdown 语法：**
```markdown
# 一级标题
## 二级标题
### 三级标题

**粗体文本**
*斜体文本*

- 列表项1
- 列表项2

[链接文本](链接地址)

![图片描述](图片路径)
```

**插入超链接：**
```markdown
# 内部链接（网站内页面）
[关于我们](/about/)
[产品服务](/products/)
[新闻动态](/news/)

# 外部链接（其他网站）
[Hexo 官方文档](https://hexo.io/zh-cn/docs/)
[GitHub Pages](https://pages.github.com/)

# 链接带标题（鼠标悬停显示）
[链接文本](链接地址 "链接标题")

# 链接新窗口打开
[链接文本](链接地址){:target="_blank"}
```

**链接使用建议：**
- 内部链接使用相对路径（如 `/about/`）
- 外部链接使用完整 URL（如 `https://example.com`）
- 重要链接添加描述性文本
- 外部链接建议在新窗口打开

**插入代码：**
```markdown
\```bash
npm install
\```

\```javascript
console.log('Hello');
\```
```

### 4. 多语言文章

- **简体中文文章**：放在 `source/_posts/` 目录
- **英文文章**：放在 `source/en/_posts/` 目录
- **繁体中文文章**：放在 `source/zh-TW/_posts/` 目录
- **日语文章**：放在 `source/ja/_posts/` 目录

**示例：**
```
source/
├── _posts/              # 简体中文文章
│   ├── 2024年度总结.md
│   └── 2025光伏趋势.md
├── en/_posts/           # 英文文章
│   └── solar-panel-launch.md
├── zh-TW/_posts/        # 繁体中文文章
│   └── solar-panel-launch.md
└── ja/_posts/            # 日语文章
    └── solar-panel-launch.md
```

### 5. 文章图片

**支持的图片格式：**
- **推荐格式**：SVG（矢量图，清晰度高，文件小）
- **常用格式**：JPG、PNG、GIF、WebP
- **图片大小**：建议单张图片不超过 2MB
- **图片尺寸**：根据使用场景调整，建议宽度 800-1200px

**图片位置：**
- 文章相关图片放在 `themes/enterprise/source/images/posts/` 目录
- 推荐使用 SVG 格式
- 文件名使用中文或英文都可以

**在文章中引用图片：**
```markdown
![图片描述](/images/posts/图片文件名.svg)
```

**图片优化建议：**
- 使用 SVG 格式获得最佳性能
- JPG/PNG 图片建议压缩后使用
- 保持图片文件名简洁，避免特殊字符
- 使用相对路径，不要使用绝对路径

### 6. 预览文章

**启动本地服务器：**
```bash
npm run server
```

**访问地址：**
- 首页：http://localhost:4000/
- 文章列表：http://localhost:4000/news/
- 单篇文章：http://localhost:4000/news/文章标题/

## 二、网站图标素材修改

### 1. 图片文件位置

- 网站图标：`themes/enterprise/source/images/` 目录
- 文章图片：`source/images/` 目录
- 主题图片：`themes/enterprise/source/images/` 目录

### 2. 主要图片文件

| 图片类型 | 位置 | 说明 |
|---------|------|------|
| 网站图标 | `themes/enterprise/source/images/hero-bg.svg` | 网站首页背景图 |
| 团队头像 | `themes/enterprise/source/images/about/team*.svg` | 团队成员头像 |
| 合作伙伴 | `themes/enterprise/source/images/partners/partner*.svg` | 合作伙伴Logo |
| 资质证书 | `themes/enterprise/source/images/about/cert*.svg` | 公司资质证书 |
| 公司图片 | `themes/enterprise/source/images/about/company.svg` | 公司外观图片 |
| 文章图片 | `themes/enterprise/source/images/posts/*.svg` | 文章相关图片 |

### 3. 修改方法

1. 替换图片文件：直接替换对应目录下的图片文件
2. 保持文件名一致：确保替换后的文件名与原文件名相同
3. 使用SVG格式：推荐使用SVG格式图片，确保清晰度和加载速度
4. 图片尺寸：保持与原图片相同的尺寸比例

## 三、网站配置修改

### 1. 基本配置

编辑 `_config.yml` 文件：

```yaml
# 网站信息
title: 网站标题
description: 网站描述
keywords: 关键词
author: 作者名

# 部署配置
deploy:
  type: git
  repo: https://github.com/Alien-Et/Alien-Et.github.io.git
  branch: gh-pages
```

### 2. 多语言配置

编辑对应语言文件：
- 简体中文：`themes/enterprise/languages/zh-CN.yml`
- 英文：`themes/enterprise/languages/en.yml`
- 繁体中文：`themes/enterprise/languages/zh-TW.yml`
- 日语：`themes/enterprise/languages/ja.yml`

### 3. 主题配置

编辑 `themes/enterprise/_config.yml` 文件：

```yaml
# 主题配置
menu:
  home: /
  about: /about/
  products: /products/
  news: /news/
  contact: /contact/
```

## 四、部署网站

### 1. 自动化部署（推荐）

**GitHub Actions 已配置自动化部署** ✅

现在推送到 main 分支后，会自动：
1. 构建网站
2. 部署到 gh-pages 分支
3. 更新 GitHub Pages 和 CF Pages

**工作流程：**
```bash
# 1. 创建/修改文章
# 编辑 source/_posts/ 中的文件

# 2. 提交到 main 分支
git add .
git commit -m "提交信息"
git push origin main

# 3. 等待自动部署完成
# 访问 https://github.com/Alien-Et/Alien-Et.github.io/actions 查看部署状态
```

### 2. 手动部署（可选）

如果需要手动部署，可以使用以下命令：

```bash
# 清理缓存
npx hexo clean

# 构建网站
npx hexo generate

# 部署到 gh-pages 分支
npx hexo deploy
```

### 3. GitHub Pages 使用

**自动部署**（推荐）：
1. GitHub Pages 已配置为使用 `gh-pages` 分支
2. 每次推送到 main 分支后自动更新
3. 访问 https://alien-et.github.io 查看网站

**手动配置**（如果需要）：
1. 登录 GitHub，进入 `Alien-Et/Alien-Et.github.io` 仓库
2. 点击 "Settings" → "Pages"
3. 在 "Build and deployment" 部分：
   - Source 选择 "Deploy from a branch"
   - Branch 选择 "gh-pages"
   - 点击 "Save"
4. 等待部署完成后，访问 `https://alien-et.github.io`

### 4. Cloudflare Pages 使用

**自动部署**（推荐）：
1. CF Pages 已配置为使用 `gh-pages` 分支
2. 每次推送到 main 分支后自动更新
3. 访问你的 CF Pages 域名查看网站

**手动配置**（如果需要）：
1. 登录 Cloudflare Dashboard
2. 进入 "Workers & Pages" → "Pages"
3. 点击 "Create a project" 或选择现有项目
4. 连接到 GitHub 仓库：
   - 选择 `Alien-Et/Alien-Et.github.io` 仓库
   - 选择 `gh-pages` 分支
   - 构建命令：留空（无需构建）
   - 输出目录：留空（使用根目录）
5. 点击 "Save and Deploy"
6. 等待部署完成后，访问你的 CF Pages 域名

## 五、常见问题

### 1. 文章不显示
- 检查文章是否在正确的目录中
- 检查Front Matter格式是否正确
- 执行 `npx hexo clean && npx hexo generate` 重新构建

### 2. 图片不显示
- 检查图片路径是否正确
- 确保图片文件存在
- 执行 `npx hexo clean && npx hexo generate` 重新构建

### 3. 部署失败
- 检查GitHub仓库地址是否正确
- 确保GitHub仓库存在且可访问
- 检查网络连接是否正常
